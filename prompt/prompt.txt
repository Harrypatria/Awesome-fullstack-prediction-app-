SECTION 2 — PROBLEM STATEMENT
────────────────────────────────────────────────────────────────────────────────

BUSINESS PROBLEM:
  Organizations lose 1.5–2x annual salary per voluntary departure. HR teams
  currently rely on exit interviews — data collected AFTER attrition occurs.
  We need a proactive system that predicts WHO will leave, WHY they will leave,
  and WHAT actions can prevent it, all surfaced in a real-time command center.

TARGET USERS:
  - Chief People Officers (CPO) needing board-level retention metrics
  - HR Business Partners executing individual intervention plans
  - People Analytics teams building evidence-based retention programs

SUCCESS METRICS:
  - Model AUC-ROC ≥ 0.85 on holdout set
  - SHAP waterfall explanations per individual prediction
  - Risk threshold alert at >70% predicted attrition probability
  - Task completion tracking from insight to resolution

  5. Export all plots to /reports/eda/ as PNG (300 DPI)

PHASE 3 — PREPROCESSING (backend/preprocessing.py)
  Steps wrapped in sklearn.pipeline.Pipeline:
  ┌──────────────────┬────────────────────────────────────────────────────┐
  │ Step             │ Transformer                                        │
  ├──────────────────┼────────────────────────────────────────────────────┤
  │ Numeric scaling  │ StandardScaler on all int/float columns            │
  │ OHE categorical  │ OneHotEncoder(drop='first', sparse=False)          │
  │ Target encoding  │ LabelEncoder on Attrition (Yes→1, No→0)           │
  │ Train/test split │ 80/20 stratified on target, random_state=42       │
  │ Imbalance fix    │ class_weight='balanced' in model params            │
  └──────────────────┴────────────────────────────────────────────────────┘

PHASE 4 — MODEL TRAINING (backend/train.py)
  Train THREE candidate models, compare, select best:
    Model A: GradientBoostingClassifier(n_estimators=200, learning_rate=0.05,
             max_depth=4, random_state=42)
    Model B: XGBClassifier(n_estimators=200, learning_rate=0.05,
             use_label_encoder=False, eval_metric='logloss', random_state=42)
    Model C: LogisticRegression(class_weight='balanced', max_iter=1000)

  Cross-validation: StratifiedKFold(n_splits=5)
  Metrics: AUC-ROC, F1-Score (macro), Precision, Recall, Accuracy

  Hyperparameter tuning (best model only):
    GridSearchCV on: n_estimators, max_depth, learning_rate, subsample

  2. ROC curve with AUC annotation
  3. Precision-Recall curve
  4. Feature importance plot (top 15 features)

FILE: backend/main.py

ROUTES:
  POST /predict
    Body: PredictionRequest (all employee feature fields, Pydantic v2)
    Returns: {

    Body: list[PredictionRequest] (max 500)
    Returns: list[PredictionResponse]

SECTION 6 — DATABASE SCHEMA (Supabase / PostgreSQL)
────────────────────────────────────────────────────────────────────────────────

FILE: supabase/migrations/001_init.sql

-- Table: employees
CREATE TABLE employees (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  employee_ref    text NOT NULL,
  department      text,
  job_role        text,
  manager_id      uuid REFERENCES employees(id),
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now()
);

-- Table: predictions
CREATE TABLE predictions (
  id                      uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  employee_id             uuid REFERENCES employees(id),
  attrition_probability   numeric(5,4) NOT NULL,
  risk_level              text NOT NULL CHECK (risk_level IN ('LOW','MEDIUM','HIGH')),
  top_risk_factors        jsonb,
  suggested_interventions jsonb,
  model_version           text,
  created_by              uuid REFERENCES auth.users(id),
  created_at              timestamptz NOT NULL DEFAULT now()
);

-- Table: alerts
CREATE TABLE alerts (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  prediction_id   uuid REFERENCES predictions(id),
  employee_id     uuid REFERENCES employees(id),
  threshold       numeric(5,4) NOT NULL,
  status          text NOT NULL DEFAULT 'OPEN' CHECK (status IN ('OPEN','ACKNOWLEDGED','RESOLVED')),
  assigned_to     uuid REFERENCES auth.users(id),
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now()
);

-- Table: tasks (Operations layer of ITDO)
CREATE TABLE tasks (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  alert_id        uuid REFERENCES alerts(id),
  employee_id     uuid REFERENCES employees(id),
  title           text NOT NULL,
  description     text,
  intervention    text,
  status          text NOT NULL DEFAULT 'TODO' CHECK (status IN ('TODO','IN_PROGRESS','DONE','CANCELLED')),
  due_date        date,
  assigned_to     uuid REFERENCES auth.users(id),
  created_by      uuid REFERENCES auth.users(id),
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now()
);

-- RLS: Enable on all tables
ALTER TABLE employees   ENABLE ROW LEVEL SECURITY;
ALTER TABLE predictions ENABLE ROW LEVEL SECURITY;
ALTER TABLE alerts      ENABLE ROW LEVEL SECURITY;
ALTER TABLE tasks       ENABLE ROW LEVEL SECURITY;

-- Indexes
CREATE INDEX idx_predictions_employee ON predictions(employee_id);
CREATE INDEX idx_predictions_risk ON predictions(risk_level);
CREATE INDEX idx_alerts_status ON alerts(status);
CREATE INDEX idx_tasks_assigned ON tasks(assigned_to);
